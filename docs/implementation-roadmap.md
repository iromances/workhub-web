# WorkHub 前端实施路线

## 1. 实施目标

前端第一阶段要交付一套可用的后台控制台，支持从待整理箱到正式工作项的完整操作闭环，而不是只做静态展示。

## 2. 里程碑

### M1. 工程基线

- 初始化 Vue 3 + TypeScript + Vite 工程
- 接入路由、状态管理、请求层、布局骨架
- 建立统一主题、通用表格、通用搜索栏、通用详情抽屉

### M2. 登录与基础框架

- 登录页
- 权限拦截
- 首页布局
- 菜单与路由骨架
- 用户信息与退出

### M3. 主业务 MVP

- 项目列表/详情
- 迭代列表/详情
- 版本列表/详情
- 工作项列表/详情
- 跟踪记录面板
- 状态流转动作面板

### M4. 收件与整理

- 企微待整理箱
- 人工新建入口
- 粘贴文本整理入口
- AI 建议结果确认页

### M5. 可视化与增强

- 看板页
- 首页工作台
- 到期提醒与通知中心
- 常用筛选和保存视图
- 支付配置管理页

## 3. 页面优先级

必须先做：

- 登录页
- 项目列表
- 工作项列表
- 工作项详情
- 待整理箱

随后补齐：

- 迭代管理
- 版本管理
- 看板
- 工作台

## 4. 开发顺序约束

1. 先接入统一布局和请求层，再做业务页面。
2. 先完成列表与详情基本闭环，再做看板和工作台。
3. 先完成人工录入和人工确认，再接 AI 结果确认。
4. 先按后端契约开发，不允许为了提速自行发明字段。

## 5. 验收基线

1. 用户可以登录并进入后台。
2. 用户可以查看项目、工作项、待整理箱。
3. 用户可以创建工作项、修改负责人、写跟踪记录、执行状态流转。
4. 用户可以查看企微原始内容和 AI 整理建议，并确认生成正式工作项。
5. 列表、详情、看板展示出的状态和后端一致。
